Lineup Recommender «Final Report»

Part 1: Overview and Hypothesis

FanDuel and DraftKings have one-day fantasy sports contests. For Basketball, the contests require the user to enter with a lineup of 9 players that are playing in the games for that day. People enter these contests everyday by paying a fixed amount of money. Once the day’s games are done, the users with the lineups containing the best performing players (calculated using a linear combination of their individual stats), win a huge amount of money. With the aim of doing well on the daily FanDuel contests, we hypothesized the following: We can attain more than 90% accuracy given enough historical data and the right selection of features. Projecting individual stats that contribute to a player’s overall performance can be better than directly projecting the fanduel score because we can choose only the relevant features for each specific stat.

Part 2: Data Collection

We are using data from 3 different sources: NBA, which returns every player’s historical data, NumberFire, which provides details about the players that are playing in today’s games, and Stattleship, which has information about each player (like height, weight, etc.). The data displayed above is after combination and cleaning which involved the following steps:

  • Create id maps to match players between the three data sources.
  • Accumulate data for the same player from different parts of each website.
  • Remove games that the player did not play in.
  • Remove bad values from gamelogs.
For every player, we collect the following information associated with that player:

Gamelogs: a list of games they have played in, where each game has:

  • Time the game was held
  • Information on whether the game was played at home or away
  • The opponent team
  • Plus minus
  • How many times the player played in that game (?)
  • Total rebounds score for the team the player is in
  • Total assists score
  • Total steals score
  • Total blocks score
  • Total turnovers score
  • Total points
  • Position: a list of positions the player has, as some players have more than one position. For instance, Derek Williams plays in Point Forward and Small Forward.

Part 3: Regression and Projection

We decided to compare 5 different kinds of regression algorithms, and compare their projections:

  • A linear regressor
  • A Lasso regressor (linear model using L1 regularization) with an alpha of 0.1
  • A random forest regressor using 1000 estimators (trees) and max splits of sqrt(#-of-features)
  • A support vector regressor with linear kernel, using a penalty coefficient of 1 and epsilon of 0.1
  • A support vector regressor with RBF kernel, using a penalty coefficient of 1 and epsilon of 0.1

Our linear, Lasso, and random forest regressors were trained on all player game data from the past 4 seasons. Our support vector regressors were trained on a single player’s game data (the player we are projecting) from the past 4 seasons. This was because SVM runtime is approximately O(#-of-observations2), forcing us to limit the number of training samples.
