Overview

As you build features, document them here!

Hopefully you have this page open in both a browser (like Chrome, Firefox, Safari, or Edge) and a text editor (like Sublime or Atom). In the browser, you should see a nicely formatted page (albeit not terribly extravagant) and in the text editor you should see a mix of plain text (like this!) and these weird pointy things called tags, like <div> and <p>.

Tags denote elements, which are individual components in a web page. They begin with a less than sign, <, and end with a greater than sign, >. This element, for example, is a paragraph. This chunk of text is surrounded by <p> and </p>. These are start and closing tags respectively. Closing tags are the same as starting tags, except they begin with a forward slash, /. Most tags have both start and closing tags. For proper HTML, you must nest your tags correctly. For example: 

            <em>It is Arthur, <strong>King of the Britons!</strong></em> 
            <em>It is Arthur, <strong>King</em> of the Britons!</strong>

Some elements are made up of only one tag, such as <img src="example.gif" alt="Alt text on hover over">. These are known as empty elements and need no closing tag. You may see some people use syntax similar to <img src="thing.jpg" />. Do not do this. It is bad and should be avoided.

If you've been following along in the source, you may also note that when the tags are displayed in text, we can't use < and >. Instead, we use ampersands to denote special characters, like the less than (<) and greater than (>) signs. They also work for other characters like ¥, £, ∞, and ≈. Additionally, if your page is encoded in Unicode (specified by <meta charset="utf-8"> in your head element) you can just type the symbols directly: ¥, £, ∞, and ≈.

Basic sturucture

A basic HTML page looks like this: 

            <html>
                <head>
                    <title>Title goes here</title>
                </head>
                <body>
                    <p>This is a basic web page!</p>
                </body>
            </html>

The entire page is wrapped in <html> tags. At the top of the source code is a <head> element, which contains metadata (the data about data) for the web page along with the page's title. After the head comes <body>, in which all of the page's content goes. Most of the time, you'll be editing content within the page's body.

Attributes and Style

Another important part of HTML is being able to style your page with CSS. CSS (Cascading StyleSheets) is used to apply style to web pages. We won't go over how to create CSS here, but we will go over how to use it. If you want to learn, and it may be helpful for visualization later, visit this page. That website, MDN, will be immensely helpful for all things HTML, CSS, and JavaScript related.

You may have noticed a few tags that have attributes, or something="somethingElse". For example, <div class="page-header center"> This div element has a class attribute with two values, page-header and center. These are provided to you by Bootstrap, which we included at the top of the page via a <link> tag. Bootstrap provides a multitude of classes and IDs which allow you to easily style your pages. The HTML stencils we provide you will all already have Bootstrap included. Classes come from CSS, and provide styling to any elements of them class. IDs are similar, except only one element should ever be of a certain ID. (Web browsers won't enforce this and it generally won't matter, but you should generally prefer classes to IDs).

Above, the page-header class denotes an element as the header of a page. By applying it to a div, everything within that div will also be recognized as part of the page-header class. The center class centers elements. Now, create a paragraph where its class is equal to lead. This should make the paragraph stand out.

Good job! You should now see how adding classes to elements can style them. You should always be enclosing your attribute values in quotes, either ' or ".

There are also some tags that provide styling. These include <mark> which highlights text, <u> for underlined text, and <small> for small text. A full list of these would be impossible to provide here. If you ever need to apply a certain style, such as bold or italic, a quick Google search should give you the answer.

At this point, you've probably seen a lot of the <code> tag if you've been following along in the source. This makes a portion of text look like source code. You'll likely wind up using this a lot during your writeups. However, it doesn't really work for multiple lines. Look:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Integer tempus lectus non dolor malesuada varius. Donec lacinia dapibus erat, ac lacinia lectus aliquet at. Curabitur a sagittis orci. Aliquam sollicitudin efficitur semper. Why are you still reading this? It's not even real Latin...

For multi-line code blocks you'll want to use the <pre> tag. Everything within a pre element will be treated as raw text. There is one catch though: unlike normal HTML which ignores whitespace of more than one space, pre will interpret all whitespace literally. For example: 

                This is somewhat undesireble, isn't it?
                I mean, look at how indented this is...

Instead, you'll want to do something like this: 

            Look at me! I'm all the way on the left!
            Aren't I awesome?

Finally, let's go over greating lists and tables. You may have to create one of each of these.

  • Lists are created with either the <ul> or <ol> tag. These are for unordered and ordered lists respectively.
  • Each item in the list is denoted by <li> for a list item.

Tables are a little more complicated. Tables begin and end with the <table> tag. If you set the class to table Bootstrap will make your table much prettier than it otherwise will be. Tables can begin with an optional <thead> row for the table's head. Next comes the table's body in <tbody>. Every row is denoted by <tr> and each element is denoted by <td> (or <th> in table headers).

Adding images

Although you may not have created a visualization, you should link to an image to learn how to do it. Use the <img> tag for an image. You'll want to provide a link to the image using src="image location". We've given you an image called example.gif. Add it to this page by creating an image tag with example.gif as its src.

Once you've done that, you may notice that the image isn't centered. That's okay! If you scroll up to the top of this page, you'll notice a <style> tag containing a class called img-responsive. Set the image's class to img-responsive and you'll notice that your image becomes centered.

We've provided a very brief, but hopefully helpful, introduction to HTML. Hopefully at this point you know enough to at least edit the stencil we give you. If you want to learn more, take a look at this MDN learning site.
